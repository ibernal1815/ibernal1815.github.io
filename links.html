Creating Links

Overview

Perhaps the most important aspect of HTML is the ability to link from one document to another. This is the basic concept underlying the web and without this functionality the World Wide Web would not exist.

Link syntax

Creating links is relatively straightforward, and the syntax provides a lot of flexibility in where and how links are applied. To create a link, you’ll use the anchor element (<a>) to wrap the content you wish to convert to a link. Attributes inside the anchor tag tell the browser where the page is linking to, or point to an external resource that the browser should download.

Take the following link example:

<a href="syntax.htm" title="learn more about syntax">HTML syntax</a>

The href attribute tells the browser how to resolve the link; that is where the user should be directed when the link is clicked. The optional title attribute provides a description of the link and can be helpful for accessibility (use sparingly — screen reader behaviour varies).

Link types

There are three basic types of link: absolute, site-root relative, and document relative. Each type refers to the value of the href attribute, which directs how the browser resolves the link once clicked.

Absolute links

Absolute links contain the full URL, including protocol. They’re typically used for external resources:

<a href="http://www.google.com" title="You can find it at google.com!">google.com</a>

Document relative links

Document-relative links navigate relative to the current document’s location. Example (same folder):

<a href="contact.htm" title="our contact page">Contact us</a>

If the target is in a subfolder:

<a href="resources/contact.htm" title="our contact page">Contact us</a>

To move up one folder level:

<a href="../contact.htm" title="our contact page">Contact us</a>

Site-root relative links

Site-root relative links always start at the site root (leading slash) and are stable regardless of the current document location:

<a href="/contact.htm" title="our contact page">Contact us</a>

Deeper in the tree:

<a href="/resources/contact.htm" title="our contact page">Contact us</a>

Fragment identifiers

Fragment identifiers let you link to a specific location within a page by pointing to an element with an id attribute. Example heading:

<h2 id="camp">Camping</h2>

Link to that fragment on the same page:

<a href="#camp">Learn more about camping</a>

Or link to a fragment on another page:

<a href="faq.htm#camp">Read our FAQ on camping.</a>

Best practices

  • Use meaningful link text (avoid "click here").
  • Prefer root-relative or consistent document-relative strategies to keep links maintainable.
  • For external links, consider rel="noopener noreferrer" when using target="_blank".
  • Keep title attributes only when they add value — they aren’t a replacement for descriptive link text.
